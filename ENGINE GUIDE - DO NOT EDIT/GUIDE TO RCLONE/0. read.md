# 🚀 Cloudflare R2 Workspace Sync Guide

This folder contains the complete toolset to sync the `zee-media-production` or any Cloudflare R2 bucket with your local workstation. Follow these steps sequentially on any fresh computer to get up and running in minutes.

---

## 🛠️ Step 1: Download & Extract Rclone

1. Go to the official download page: **rclone.org/downloads**
2. Download the **Intel/AMD - 64 Bit** version for Windows (or the variant matching your OS architecture).
3. Open the downloaded `.zip` file and extract **only** the `rclone.exe` file directly into this folder.
   - _Note: Your folder contents must look like this before proceeding:_
     - `0. read.md`
     - `install_rclone.bat`
     - `1. Download_Media.bat`
     - `2. Upload_Media.bat`
     - `rclone.exe` 👈 _(Must be present for installation!)_

---

## ⚙️ Step 2: One-Click System Installation

Before you can run the sync scripts, Windows needs to recognize Rclone globally.

1. **Right-click** on `install_rclone.bat`.
2. Select **"Run as administrator"**.
3. This script will automatically:
   - Create a permanent system directory at `C:\Rclone`.
   - Move `rclone.exe` into that folder.
   - Inject it into your Windows System Environment variables so it works anywhere.
4. Once the terminal prints `SUCCESS`, press any key to close it.

---

## 🔑 Step 3: Link Your Cloudflare Credentials

You must authenticate your machine with Cloudflare. You only need to do this **once** per computer.

1. Open your Windows Start Menu, type **`cmd`**, and press **Enter** to open a fresh Command Prompt.
2. Type **`rclone config`** and press **Enter**.
3. Follow this exact text-based menu path:
   - Type **`n`** for _New Remote_ and press Enter.
   - Name it exactly: **`cloudflare`**
   - For storage type, scroll or type **`s3`** _(Amazon S3 Compliant Storage)_ and press Enter.
   - For provider, scroll or type **`Cloudflare`** and press Enter.
   - Choose option **`1`** to enter your credentials manually.
   - **`access_key_id`**: _Paste your alphanumeric Cloudflare R2 Access Key ID._
   - **`secret_access_key`**: _Paste your long Cloudflare R2 Secret Access Key._
   - **`endpoint`**: _Paste your unique Cloudflare URL:_ `https://<YOUR_ACCOUNT_ID>.r2.cloudflarestorage.com`
4. Leave all remaining settings entirely blank by pressing **Enter** through the prompts until it confirms and saves.
5. Close the Command Prompt window.

---

## 🔄 Step 4: Daily Workflows (How to Sync)

Now that the system is configured, you never have to touch the command line again. Use these two scripts on your desktop or inside this folder for your day-to-day work:

### 📥 1. Download_Media.bat

- **What it does:** Clones your entire R2 cloud bucket down to your local machine (`D:\Workspace\zee-media-production\R2_Bucket_Media`).
- **Why it's smart:** It uses background delta checks to skip files you already have, saving your internet bandwidth and Cloudflare API operation limits.
- **When to use it:** Run this at the start of your workday to make sure you have the absolute latest media files downloaded locally.

### 📤 2. Upload_Media.bat

- **What it does:** Copies new local files up to the Cloudflare R2 bucket.
- **Why it's safe:** It uses `copy` instead of `sync`. This ensures that if you accidentally deleted a file on your local hard drive, it **will not** delete it from your cloud bucket safety backup.
- **When to use it:** Run this at the end of your workday or after finishing a major media production batch to push your changes safely to the cloud.

### 🚨 3. Force_Sync_Up_Cloud_Clean.bat

- **What it does:** Forces your Cloudflare R2 bucket to mirror your local drive. This script will permanently delete files from your cloud storage if you deleted them locally.
- **Double-Safety Mechanism:** When clicked, it automatically runs an initial **--dry-run simulation**. It lists every file that _would_ be uploaded or deleted without actually doing it.
- **When to use it:** Use this when cleaning clutter out of your workspace. Inspect the initial simulation readout on the screen carefully. If the list of projected deletions looks correct, type `RUN` to execute. If it looks wrong, type `N` to safely abort.

## ⚠️ Maintenance & Troubleshooting

- **`rclone is not recognized...` Error:** This means `install_rclone.bat` wasn't run as an Administrator, or your system terminal needs to be restarted to refresh its environment path paths.
- **Modifying Folders:** If your local workspace paths change in the future, right-click either `.bat` script, select **Edit**, and update the `"D:\Workspace\..."` string inside the double quotes.
