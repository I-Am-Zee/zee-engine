# Mistakes Log — 2026-05-02 13:11

## The Mistake
I applied the Tailwind CSS v4 **HTML-specific** importance modifier (`!`) inside a CSS file's `@utility` block.

**Invalid Code:**
```css
@utility motion-soft {
  transition-property: all!;
  transition-duration: 500ms!;
}
```

## Why It Happened
I hallucinated that the new Tailwind v4 "postfix important" rule applied to the CSS declaration layer. I confused **utility class usage** in HTML with **utility definition** in CSS.

## The Consequence
1. **Invalid CSS**: The browser does not recognize the `!` suffix in a CSS property value.
2. **Utility Failure**: The entire `.motion-soft` class was discarded by the browser's CSS parser.
3. **Motion Regression**: The `GlassProductCard` lost its hover animations entirely because the "Engine" utility was technically non-existent.

## The Correction
Replaced the `!` suffix with the standard CSS `!important` keyword.

**Correct Code:**
```css
@utility motion-soft {
  transition-property: all !important;
  transition-duration: 500ms !important;
}
```

## The Lesson (Hardened Rule)
- **HTML/JSX Layer**: Use the `!` suffix (e.g., `flex!`, `bg-red-500!`).
- **CSS Layer (@utility/global.css)**: Use standard `!important` (e.g., `display: flex !important`).
- **Verification**: Always verify that "new" syntax features for one layer (HTML) don't bleed into another (CSS) unless explicitly documented as a cross-layer feature.
