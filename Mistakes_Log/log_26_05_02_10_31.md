# Mistakes Log — 2026-05-02 10:31

## 1. Aesthetic Over-Reach (Forced Hover Colors)
- **Error**: Added `group-hover:text-accent-brass` to the Featured BlogCard title without explicit instruction.
- **Cause**: Assumed the brand accent color should be the primary interaction feedback, ignoring the user's preference for a pure monochromatic "Old Money" aesthetic.
- **Impact**: Broke the visual consistency of the "Cover Story" and required a manual rollback of the hover state.
- **Lesson**: Interactivity in premium designs should rely on motion and depth (zoom, translation) rather than color shifts unless explicitly requested.

## 2. Structural Assumption (Section Merging)
- **Error**: Merged the Page Header and Featured Hero into a single `<Section variant="airy">`.
- **Cause**: Attempted to "simplify" the component tree by grouping top-level elements, which inadvertently changed the vertical rhythm and alignment.
- **Impact**: Lost the sharp architectural separation between the Journal's title and its featured content, requiring a revert to two distinct `<Section>` blocks.
- **Lesson**: Maintain atomic section boundaries. Top-level layout sections should remain discrete to allow for independent padding and alignment control.

## 3. Premature Schema Modification
- **Error**: Attempted to solve a UI contrast issue by adding a `theme` field to the global blog collection schema.
- **Cause**: Mistook a "Page Layout" problem for a "Content Data" problem.
- **Impact**: Polluted the CMS schema and would have forced unnecessary metadata on every future post.
- **Lesson**: UI toggles belong in Page settings or Props. Never modify the project's "DNA" (schema) to solve a one-off visual requirement.

## 4. Partial Implementation (Incomplete Style Flipping)
- **Error**: When instructed to make "everything white" on the Featured card, I initially only flipped the Heading and Excerpt, missing the Author name, Date, and Metadata.
- **Cause**: Failed to do a full visual audit of the component's sub-elements before applying the edit.
- **Impact**: Resulted in a "half-finished" look with mixed black and white text, requiring a second corrective pass.
- **Lesson**: "Everything" means 100%. Always perform a recursive audit of a component's primitives (Text, Badge, Icon) when applying a global style change.

## 5. Technical Execution (Replacement Chunk Failures)
- **Error**: Attempted multiple code replacements using `multi_replace_file_content` that failed because the target content strings were slightly off.
- **Cause**: Inaccurate copying of line indentation or whitespace from the source file.
- **Impact**: Slowed down the development rhythm and required repeated `view_file` calls to verify the exact state of the code.
- **Lesson**: Double-check whitespace and line endings in target strings, especially in files with complex indentation like Astro templates.
