# Extensive Audit & Visual Inspection Report
**Date:** May 15, 2026

## 1. Visual QA Findings (via Chrome DevTools MCP)
- **Home Page Hero:** The Hero section on `localhost:4321` renders correctly with `<Section variant="hero">` but the image has `opacity-60` with a heavy white gradient overlay making it look slightly washed out. This seems intentional for the "Permanent Glow" aesthetic.
- **Announcement Bar:** The announcement bar uses `bg-surface` and `text-text-main` as fallbacks, but the CMS settings might be injecting faint colors. The text is barely visible on the Hero's white background.
- **Breadcrumbs (Shop & Collections):** Non-slug pages (like `/shop` index) have the breadcrumb component loaded, but they aren't visually aligned with the strict `max-w-7xl px-4 lg:px-8` content containers because `Breadcrumbs.astro` is using a raw `<nav>` and the internal padding is throwing off the left alignment relative to the grid. The recent change to add `snap-x` and `snap-start` works, but the underline on active links might have been broken during a Tailwind v4 token sweep.

## 2. Architectural Violations (Primitive Usage Contract)
As per Atomic Design methodology and the Primitive Usage Contract (`@Project_Skill/zelia-vance-engine.md`), components MUST NOT bypass primitives (`Section`, `Heading`, `Text`) by using raw HTML tags with hardcoded Tailwind classes.

### **Layout Violations (Bypassing `<Section>`)**
- `src/components/ui/Breadcrumbs.astro`: Uses a raw `<nav>` and an inner `<div>` with hardcoded `mx-auto max-w-7xl px-4 lg:px-8` instead of leveraging `<Section as="nav" variant="standard" containerClass="...">` or similar.
- `src/components/ui/EditorialHero.astro`: Uses raw `<div>` with hardcoded `max-w-7xl px-6 sm:px-8 lg:px-12` and manual padding `py-14 md:py-24`.
- `src/components/features/LookbookCarousel.astro`: Uses a raw `<section>` with `py-32 px-4 lg:px-8` instead of the `<Section>` primitive.
- `src/components/features/CompleteTheLook.astro`: Uses `<section class="... pt-12 md:pt-16 pb-8 md:pb-12 px-4 lg:px-8 max-w-7xl mx-auto">`.
- `src/components/features/RelatedProducts.astro`: Uses `<section class="... pt-12 md:pt-16 pb-8 md:pb-12 px-4 lg:px-8 max-w-7xl mx-auto">`.
- `src/pages/brand/[slug].astro` & `src/pages/legal/[slug].astro`: (Based on prior logs) Using raw `<article>` and `<section>` tags with "magic number" padding instead of `<Section variant="standard">`.

### **Typography & Token Violations (Bypassing `<Text>` / Tailwind v4)**
- `src/components/ui/Breadcrumbs.astro`: The separator (`/`) is a raw `<span>` with `text-[10px]`. It should be `<Text variant="metadata">`. Also, active links use raw `text-xs uppercase tracking-widest` instead of mapping to `<Text variant="metadata" color="main" weight="bold">`.
- `src/components/ui/EditorialHero.astro`: The bullet separators (`•`) are raw `<span>` tags.
- `src/components/ui/BlogCard.astro`: The "Words by" section uses raw `<span>` separators (`·`) instead of `<Text variant="metadata" color="muted">`.
- `src/components/features/FreeShippingNudge.astro`: Uses raw `<p>` and `<span>` tags with `text-[0.8125rem]` and `text-[0.6875rem]` instead of `<Text variant="caption">` or `<Text variant="metadata">`.
- `src/components/features/AffiliateQuickView.astro`: Uses raw `<span>` for pricing (`text-2xl font-bold tracking-tight`) instead of `<Text variant="price">`. The tags list also uses raw `<span class="text-xs font-medium">` instead of `<Text variant="tag">`.
- **Tailwind v4 Token Bleed:** While some tokens were fixed (e.g., `bg-(--color-surface)`), many components like `BlogCard.astro`, `Footer.astro`, and `NewsletterForm.astro` still have a mix of raw spacing and text utility classes instead of fully conforming to the `(--token)` syntax required by Vite/Tailwind v4.
